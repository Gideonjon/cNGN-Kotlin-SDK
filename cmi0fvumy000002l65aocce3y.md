# **How to Start Contributing & Winning on NEAR (Step-by-Step)**

Here‚Äôs the exact blueprint I used to go from zero to winning on NEAR.

1. ## **Pick a Problem NEAR Needs Right Now**
    

Don‚Äôt start with:

*‚ÄúI want a grant.‚Äù*

Start with:

*‚ÄúWhat does NEAR need that I can build?‚Äù*

Examples of problems you can solve by building on NEAR includes:

* **Kotlin/Android SDK:**
    

Many devs want native Android access to [NEAR Remote Procedure Call (RPC)](https://docs.near.org/api/rpc/introduction) without needing JavaScript.

* **Wallet SDK for mobile apps:**
    

Native library to integrate NEAR login or transactions into Android apps.

* **Kotlin OpenAPI clients:**
    

Auto-generated clients for interacting with NEAR RPC, account lookup, indexing, or specific contract APIs.

* **Mobile-first dApps**
    

Decentralized mobile apps habving on-chain functionalities.

* **Tools for onboarding:**
    

QR connect, key management UI, signing UI components tools.

* A Kotlin NEAR RPC wrapper
    
* A sample Kotlin dApp
    
* A wallet connect demo
    
* A NEAR Java/Kotlin library for beginners
    

**Publish tutorials**

* ‚ÄúHow to build a NEAR wallet on Android‚Äù
    
* ‚ÄúUsing OpenAPI to generate NEAR clients in Kotlin‚Äù
    
* ‚ÄúHow to sign transactions on NEAR from Android‚Äù
    

**Submit small PRs to NEAR repos**

* Find fixes to make.
    
* Docs to contribute
    
* Examples to make.
